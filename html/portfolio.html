minitalk 120/100

Summary: The purpose of this project is to code a small data exchange program using UNIX signals. As a bonus, the program also handles UNICODE characters and provide an acknowledgement mechanism so that the server can inform the client when it has received a complete message.
Skills: C, UNIX, signals, systems programming.
Notes: The project makes you send a string from a client to a server by encoding each character binary value using SIGUSR1 and SIGUSR2. While this was a fun little educative project, it is to be noted that signals are not meant to be used to transmit data. Their purpose is to allow a program to inform another program that an event occured. For example, when a user press Ctrl-C, the OS sends the SIGINT signal to the program currently running in the foreground to inform it that should INTerrupt itself i.e exit.

Notes: The library contains a lot of functions of questionnable utility and lack some useful ones.
